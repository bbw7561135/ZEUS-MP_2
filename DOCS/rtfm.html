Register as a ZEUS-MP User

If you would like to receive emailed notices of code revisions, upgrades, and bug fixes, please register as a ZEUS-MP user via this simple process:

  • Address an email to John Hayes.
  • Type "add ZEUS-MP user" in the subject heading.

Your email address will be added to a mailing list. Your email address will not be shared with anyone under any circumstances.

Feedback

It is the wish of myself and the LCA that ZEUS-MP be a useful code to you. Additionally, it is my wish that the documentation on this web site help you get started running ZEUS-MP with a minimum of fuss. Toward that end, your feedback on this website is welcomed and encouraged. Support for this code will be strictly limited by the guidelines I provide below (I'm not paid to do code support; this is strictly pro bono work), but within those constraints I will do my best to answer questions and be helpful.

Bug Reporting and Support Guidelines

NOTE: Support is provided ONLY for the ZEUS-MP code distributed from this Web site, beginning with version 2.0.0 and future updates. NO ASSISTANCE WHATSOEVER will be provided for the following codes with very similar names:

  1. Version 1 of ZEUS-MP (e.g. zeusmp10b, or any descendant therefrom).
  2. The ZEUS-2D code.
  3. The ZEUS-3D code.

Problems that qualify for support as indicated in the guidelines below must be reported via email to John Hayes.

To HELP ENSURE THAT YOUR EMAIL IS ACTUALLY READ: please include "ZEUS-MP Help" in the subject heading. I ask this only because, like you, I get amazing amounts of SPAM email, and a recognizable email header will help ensure that your email doesn't accidently get thrown out with the junk.

ASSISTANCE WILL BE PROVIDED IN THE FOLLOWING AREAS:

  1. Obtaining the tar file from the Web server.
  2. Suspected bugs in the source code.
  3. Customization of problem-generator subroutines provided in the tar file.

Reports of suspected bugs MUST include the following information:

  1. The platform on which the code was compiled.
  2. A copy of the Makefile.
  3. A copy of the problem generator subroutine (e.g. blast.F).
  4. A copy of the zmp_inp file.
  5. The exact command used to execute zeusmp.x.
  6. All error messages reported to the user.

Assistance in the following areas WILL NOT BE PROVIDED UNDER ANY CIRCUMSTANCES:

  1. General questions about MPI, HDF, or Fortran.
  2. Requests for additional physics/numerical capabilities.
  3. Requests for additional language/library support.
  4. Debugging problem generators not included in ZEUS-MP tar file.
  5. Your modifications to the source code.
